JTux Workstation Installation Guide

This document describes how to install JTux Workstation on a client machine in your local Tuxedo environment.

Prerequisites

Before you start the installation, make sure the following prerequisites are met:

  1. Tuxedo/WS version 8.1 is installed on the target machine.

  2. There is a Java SDK (JDK) version 1.4 or higher installed on the target machine. The JDK's java application launcher must be in the current PATH. These requirements can be checked by executing the following command: 

    java -version
    This command should succeed and display a Java version 1.4 or higher.

    If there is no suitable JDK installed on the target machine then one of the following JDKs should be installed. For Solaris, Windows and Linux you can use Sun J2SE 1.4 or higher. For Windows and Linux you can also use BEA JRockit. For HP-UX you can use HP J2SE 1.4 or higher.

  3. You have the JTux Workstation installation packages that matches the type of the target machine, i.e.
    • JTuxWS-1.1.x-win32-install.jar for Windows/Intel with 32-bit Tuxedo.
    • JTuxWS-1.1.x-linux32-install.jar for Linux/Intel with 32-bit Tuxedo.
    • JTuxWS-1.1.x-sol32-install.jar for Solaris/Sparc with 32-bit Tuxedo.
    • JTuxWS-1.1.x-sol64-install.jar for Solaris/Sparc with 64-bit Tuxedo.
    • JTuxWS-1.1.x-hpux32-install.jar for HP-UX/PA-RISC with 32-bit Tuxedo.
    • JTuxWS-1.1.x-hpux64-install.jar for HP-UX/PA-RISC with 64-bit Tuxedo.
    where x is the latest available patch level for JTux Workstation 1.1.

Installing the JTux Workstation software package

The JTux Workstation software package can be installed by running the following command: 

java -jar JTuxWS-1.1.x-platform-install.jar -dir install_dir
where install_dir is the directory in which to create the JTux Workstation product directory. For example, JTux Workstation 1.1.0 can be installed on a Solaris/Sparc machine with 64-bit Tuxedo using the following command: 
java -jar JTuxWS-1.1.0-sol64-install.jar -dir /opt
This will create a new JTuxWS-1.1.0 product directory inside the /opt directory.
