JTux.jar

JTux.jar is the Java class library containing the Java API for Tuxedo described in the JTux Programming Reference. The JTux.jar file is located in the lib subdirectory of the JTux directory.

Compiling JTux clients and servers using JTux.jar

To compile a JTux client or server using JTux.jar, simply add the JTux.jar file to the classpath of the Java compiler. For example, the command for compiling the SimpleServer class written in Java looks like this: 

javac -classpath JTUXDIR/lib/JTux.jar -d dir SimpleServer.java
where JTUXDIR is the JTux directory and dir is the output directory in which to write the SimpleServer.class file.

Running JTux clients using JTux.jar

JTux clients using JTux.jar can be run using the standard Java application launcher. JTux servers, on the other hande, must be run using the JServer executable.

To run a JTux client using JTux.jar, the JTux.jar class library must be present in the class path of the Java application launcher. Furthermore, at runtime, JTux.jar uses a native code library called JTuxInterop and the directory containing this library must be passed to the Java application launcher using the java.library.path system property. On Unix, this is the lib and on Windows the bin subdirectory of the JTux directory. Finally, to run a JTux client using a 64-bit version of JTux, the -d64 command line option must be passed to the Java application launcher as well.

For example, the following command runs the SimpleClient sample application using a 64-bit version of JTux on Unix: 

java -d64 -classpath JTUXDIR/lib/JTux.jar:dir -Djava.library.path=JTUXDIR/lib SimpleClient "Hello, World!"
and the following command runs the SimpleClient sample application using a 32-bit version of JTux on Windows: 
java -classpath JTUXDIR\lib\JTux.jar;dir -Djava.library.path=JTUXDIR\bin SimpleClient "Hello, World!"
In these examples, JTUXDIR is the JTux directory and dir is the directory containing the SimpleClient.class file.

These examples assume that the Tuxedo system environment variables are set as described in the Tuxedo installation guide (Chapter 6 "Performing Post-Installation Tasks", sections "Setting Environment variables on a Windows System" and "Setting Environment variables on a UNIX system").
